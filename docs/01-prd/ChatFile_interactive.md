## 一、全局布局

### 1.1 整体结构

采用**左右分栏**布局，响应式适配：

```plain&#x20;text
┌──────────────────────────────────────────────────────────┐
│  🧠 ChatFile                            [主题切换] [？]  │
├───────────────────────────┬──────────────────────────────┤
│                           │                              │
│     💬 对话区（左栏）       │     📖 PDF 预览区（右栏）     │
│        flex: 4            │        flex: 6               │
│                           │                              │
├───────────────────────────┴──────────────────────────────┤
│  [📎 上传/更换文档]  [  输入框 ...                ] [➤]  │
└──────────────────────────────────────────────────────────┘
```

* **默认比例**：对话区 40% / 预览区 60%

* **分栏拖拽**：中间分割线支持左右拖拽调整宽度，最小宽度各 30%

* **折叠模式**：预览区右上角 `[◀ 折叠]` 按钮可将预览区收起，对话区独占全宽；再点击 `[▶ 展开]` 恢复



### 1.2 顶部导航栏

```plain&#x20;text
🧠 ChatFile    📄 XYZ_Datasheet_v2.0.pdf (128页, 45.2MB)   [❓帮助]
```

* 左侧：产品 Logo + 名称

* 中间：当前文档名、页数、文件大小（未上传时显示"暂未加载文档"）

* 右侧：帮助入口





## 二、文档上传交互

### 2.1 空状态（首次进入 / 无文档）

整个页面居中显示上传引导区域：

```plain&#x20;text
┌─────────────────────────────────────────┐
│                                         │
│        📄                               │
│    拖拽 PDF 文件到此处                    │
│    或 点击选择文件                        │
│                                         │
│    支持 PDF 格式，单文件 ≤ 100MB          │
│                                         │
│         [ 选择文件 ]                     │
│                                         │
└─────────────────────────────────────────┘
```



**交互细节：**

* 支持**拖拽上传**：拖入文件时，边框变为蓝色虚线 + 呼吸动效

* 支持**点击上传**：点击按钮或区域，打开系统文件选择器

* **格式校验**：选择非 PDF 文件 → Toast 提示"仅支持 PDF 格式文件"，3s 自动消失

* **大小校验**：> 100MB → Toast 提示"文件大小超过 100MB 限制"



### 2.2 上传中状态

文件选定后，空状态区域变为进度展示：



```plain&#x20;text
┌─────────────────────────────────────────┐
│                                         │
│  📄 XYZ_Datasheet_v2.0.pdf              │
│                                         │
│  ████████████░░░░░░░░  65%  上传中...    │
│                                         │
│             [ 取消上传 ]                 │
│                                         │
└─────────────────────────────────────────┘
```

* 显示文件名 + 实时上传百分比进度条

* `[取消上传]` 按钮 → 二次确认弹窗："确定取消上传？" → 确认后回到空状态



### 2.3 解析处理状态

上传完成后，进入解析阶段，展示**步骤条**：



```plain&#x20;text
┌─────────────────────────────────────────┐
│                                         │
│  📄 XYZ_Datasheet_v2.0.pdf              │
│                                         │
│  ✅ 上传完成  →  🔄 文档解析中...         │
│               →  ○ 智能切片              │
│               →  ○ 向量索引              │
│                                         │
│  预计耗时：约 2 分钟                      │
│                                         │
│        [ 取消处理 ]                      │
│                                         │
└─────────────────────────────────────────┘
```



**步骤流转：**

1. `✅ 上传完成` → `🔄 文档解析中...` → `✅ 解析完成`

2. `🔄 智能切片中...` → `✅ 切片完成（共 N 个切片）`

3) `🔄 向量索引构建中...` → `✅ 索引就绪`



**交互细节：**

* 当前执行步骤显示旋转 loading 图标 + 动态省略号

* 已完成步骤显示 ✅ 绿色对勾

* 未开始步骤显示 ○ 灰色圆圈

* 展示预估剩余时间（基于文档页数估算）

* 右侧 PDF 预览区此时可以**提前加载 PDF 预览**（解析与预览并行），让用户等待时可以先浏览文档



### 2.4 解析就绪 → 进入对话

全部步骤完成后：

* 步骤条收起为顶部横幅：`✅ 文档就绪 | 128 页 | 356 个切片 | 耗时 1 分 42 秒`，3s 后自动淡出

* 页面自动切换为**分栏布局**（左：对话区，右：PDF 预览区）

* 对话区显示欢迎内容（见第三节）

* 输入框激活，获得焦点



### 2.5 解析失败

```plain&#x20;text
┌─────────────────────────────────────────┐
│                                         │
│  ❌ 文档解析失败                         │
│                                         │
│  原因：文件已加密，无法提取文本内容        │
│                                         │
│  [ 重新上传 ]    [ 查看详情 ]            │
│                                         │
└─────────────────────────────────────────┘
```

* 展示具体失败原因（加密 / 超时 / 乱码 / 文件损坏）

* `[重新上传]`：回到空状态

* `[查看详情]`：展开显示错误日志摘要



### 2.6 更换文档

对话状态下，底部栏 `[📎]` 按钮点击后弹出选项：

* **上传新文档**：二次确认"更换文档将清空当前对话，确定？" → 确认后回到上传流程

* **不做操作**：点击外部区域关闭

**交互细节：**

| 元素                 | 交互行为                                                             |
| ------------------ | ---------------------------------------------------------------- |
| `[1]` `[2]` 行内引用标记 | 蓝色超链接样式。**悬停**：Tooltip 显示切片摘要前 80 字。**点击**：右侧 PDF 预览区滚动到对应页并高亮段落 |

| 👎 点踩              | 点击后图标变实心红色，滑出反馈标签选择栏（见 3.5 节）                                    |
| 📋 复制              | 点击复制回答纯文本（不含引用标记），Toast 提示"已复制到剪贴板"                              |
| Markdown 渲染        | 支持加粗、代码块（含语法高亮）、表格、列表的完整 Markdown 渲染                             |



### 3.3 流式输出（打字机效果）

**进入生成阶段的完整时序：**



```plain&#x20;text
用户点击发送
    ↓
[1] 输入框禁用，发送按钮变为 ■ 停止按钮
    ↓
[2] 对话区底部出现状态提示：
    "🔍 正在检索相关内容..."  （检索阶段，约 0.3-1s）
    ↓
[3] 状态变为：
    "✍️ 正在生成回答..."     （生成阶段）
    ↓
[4] AI 消息气泡出现，文字逐 Token 渲染
    - 光标闪烁在文字末尾
    - 引用标记 [1] 渲染为灰色占位，生成完毕后变为可点击蓝色链接
    ↓
[5] 生成完毕：
    - 引用来源块淡入展示
    - 反馈按钮（👍👎📋）淡入展示
    - ■ 停止按钮恢复为 ➤ 发送按钮
    - 输入框恢复激活，自动获取焦点
```



**中断生成：**

* 用户点击 `[■ 停止]` → 立即停止输出，已生成内容保留在气泡中

* 气泡底部标注"⚠️ 回答已中断"灰色文字

* 引用来源基于已生成内容展示（可能不完整）



### 3.4 多轮对话

* 对话消息按时间从上到下排列，自动滚动到最新消息

* 新消息出现时：如果用户已手动上滚查看历史，不强制跳到底部，而是在底部浮现 `[↓ 新消息]` 按钮，点击跳转

* 支持上下文连续追问：

  ```plain&#x20;text
  用户：CTRL_REG 的复位值？
  AI：复位值为 0x00 [1]...
  用户：它的位域定义呢？        ← 指代"CTRL_REG"
  AI：CTRL_REG 的位域定义如下...  ← 成功消解指代
  ```

* 对话历史窗口保留最近 **10 轮**（20 条消息），超出后旧消息灰化折叠

* 折叠区域顶部显示 `[↑ 查看更多历史 (共 N 轮)]`，点击展开加载



### 3.5 反馈交互

点击 👎 后，气泡底部滑出反馈面板：

**交互细节：**

## 四、PDF 预览区交互

### 4.1 基础预览



```plain&#x20;text
┌──────────────────────────────────────┐
│  📖 文档预览           [◀ 折叠]      │
├──────────────────────────────────────┤
│  ┌──────────────────────────────┐   │
│  │                              │   │
│  │         PDF 渲染内容          │   │
│  │                              │   │
│  │   ┌─────────────────────┐    │   │
│  │   │  ███ 高亮区域 ███    │    │   │
│  │   └─────────────────────┘    │   │
│  │                              │   │
│  └──────────────────────────────┘   │
│                                      │
├──────────────────────────────────────┤
│  [−] [+]  75%   │  < 37 / 128 >    │
│  缩放控制         │  页码导航         │
└──────────────────────────────────────┘
```



**基础操作：**

| 操作   | 交互方式                                              |
| ---- | ------------------------------------------------- |
| 翻页   | 鼠标滚轮连续滚动 / 底部 `< >` 翻页箭头 / 点击页码输入框直接跳转            |
| 缩放   | `[−]` `[+]` 按钮步进 25% / Ctrl + 滚轮 / 缩放百分比点击后输入自定义值 |
| 缩放范围 | 50% \~ 300%，默认自适应宽度（Fit Width）                    |
| 页码导航 | 底部居中显示 `当前页 / 总页数`，点击页码可弹出输入框直接跳转                 |



### 4.2 引用跳转与高亮

**核心交互链路：点击引用 → 跳转 → 高亮**

```plain&#x20;text
用户点击对话区的 [1] 引用标记
         ↓
PDF 预览区平滑滚动到对应页码（scroll-behavior: smooth）
         ↓
目标段落出现黄色高亮背景（#FFF3CD），带呼吸闪烁动画 × 3次
         ↓
3 秒后高亮从亮黄渐变为浅黄（#FFF9E6）持续保持
         ↓
用户点击其他引用 → 旧高亮清除，新高亮显示
```



**高亮规则：**

* 高亮区域为切片对应的文本范围（按坐标定位）

* 若切片跨页，自动跳转到切片起始页，高亮跨页内容

* 页面左侧显示黄色竖线标记，方便滚动时快速识别高亮位置

* 同时存在多个引用时，高亮颜色区分：`[1]` 黄色 / `[2]` 蓝色 / `[3]` 绿色（最多 5 色循环）



### 4.3 迷你目录导航（侧边栏）

PDF 预览区左上角提供 `[☰ 目录]` 按钮：



```plain&#x20;text
┌──── 📑 文档目录 ─────────┐
│                          │
│  ▼ 1. Overview           │
│    1.1 Features          │
│    1.2 Applications      │
│  ▶ 2. Pin Configuration  │
│  ▶ 3. Electrical Specs   │
│  ▶ 4. Functional Desc    │
│  ▼ 5. Register Map  ←📌  │
│    5.1 CTRL_REG          │
│    5.2 STATUS_REG        │
│  ▶ 6. Timing Diagrams   │
│                          │
└──────────────────────────┘
```



**交互细节：**

* 点击 `[☰ 目录]` 按钮，目录面板从左侧滑入覆盖在 PDF 预览上方（半透明背景遮罩）

* 当前阅读位置在目录中**高亮标记**（📌 图标或蓝色背景）

* 点击任意目录项 → PDF 跳转到对应章节 → 目录面板自动收起

* 多级目录支持折叠展开（▶ / ▼）



### 4.4 预览区状态

| 状态    | 表现                    |
| ----- | --------------------- |
| 文档未上传 | 灰色占位区，居中文案"上传文档后在此预览" |
| 文档加载中 | 骨架屏占位 + 顶部加载进度条       |
| 加载完成  | 正常显示 PDF 内容           |
| 加载失败  | 居中错误提示 + `[重试]` 按钮    |



### 4.5 折叠与展开

* 点击 `[◀ 折叠]`：预览区收起为右侧窄条（约 40px），对话区全宽展开，窄条显示 `[▶]` 展开按钮

* 点击 `[▶ 展开]`：预览区恢复为默认宽度，带滑出动画（300ms ease-in-out）

* 折叠状态下点击对话中的引用标记 → 自动展开预览区并跳转到对应页



## 五、输入区交互

### 5.1 输入框

```plain&#x20;text
┌──────────────────────────────────────────────────┐
│ [📎]  [  请输入问题...                     ] [➤] │
└──────────────────────────────────────────────────┘
```



**交互细节：**

| 元素         | 行为                                                         |
| ---------- | ---------------------------------------------------------- |
| 输入框        | 单行输入，内容超长时水平滚动。按 `Enter` 发送，`Shift + Enter` 换行（最多扩展 4 行高度） |
| 占位符        | 文档就绪前："请先上传文档..."（灰色）；就绪后："输入问题，开始与文档对话..."（灰色）            |
| `[📎]` 按钮  | 上传/更换文档入口（见 2.6 节）                                         |
| `[➤]` 发送按钮 | 输入框为空时灰色不可点击；有内容时变为蓝色可点击                                   |
| 停止按钮       | AI 生成过程中，`[➤]` 变为红色 `[■]` 停止按钮                             |
| 字数限制       | 单次输入上限 2000 字，超限时输入框右下角显示红色字数统计 `2001/2000`                |



### 5.2 输入框状态流转

```plain&#x20;text
                    ┌───────────┐
     文档未就绪 ──→  │  禁用态    │  输入框灰色，不可聚焦
                    └───────────┘
                         │ 文档就绪
                         ▼
                    ┌───────────┐
                    │  空闲态    │  可输入，发送按钮灰色
                    └───────────┘
                         │ 输入文本
                         ▼
                    ┌───────────┐
                    │  可发送态   │  发送按钮蓝色高亮
                    └───────────┘
                         │ 点击发送 / 按 Enter
                         ▼
                    ┌───────────┐
                    │  生成中态   │  输入框禁用，显示停止按钮
                    └───────────┘
                         │ 生成完成 / 点击停止
                         ▼
                    ┌───────────────┐
                    │  空闲态（聚焦） │  自动清空输入框，获取焦点
                    └───────────────┘
```

## 六、异常与边界状态

### 6.1 网络/服务异常

**场景一：检索或生成服务不可用**

```plain&#x20;text
  🤖  ┌────────────────────────────────────────┐
      │ ⚠️ 服务暂时不可用，请稍后再试。          │
      │                                        │
      │ 错误码：503 | 请求ID：req_a8f3c2        │
      │                                        │
      │          [ 🔄 重试 ]                    │
      └────────────────────────────────────────┘
```

* `[🔄 重试]` 点击后重新发送上一次用户消息，消息气泡保持不变

* 连续 3 次失败 → 提示"服务持续异常，请联系管理员"，禁用输入框 30s



**场景二：生成过程中断连**

* 已输出的文本保留

* 气泡底部标注"⚠️ 网络中断，回答不完整"

* 显示 `[🔄 重新生成]` 按钮



**场景三：请求超时（30s 无响应）**

* 自动中止请求

* 提示"⏱️ 响应超时，请简化问题后重试"



### 6.2 文档相关异常

| 场景             | 提示内容                         | 操作                  |
| -------------- | ---------------------------- | ------------------- |
| 上传加密 PDF       | "该 PDF 文件已加密，无法解析"           | 提供 `[重新上传]`         |
| 上传扫描件 PDF（纯图片） | "检测到该 PDF 为扫描件，当前版本暂不支持 OCR" | 提供 `[重新上传]`         |
| 解析超时（> 5 分钟）   | "文档解析超时，文件可能过大或结构复杂"         | 提供 `[重试]` `[重新上传]`  |
| 切片数为 0         | "未能从文档中提取有效内容"               | 提供 `[重新上传]`         |
| PDF 预览加载失败     | 预览区显示"文档预览加载失败"              | 提供 `[重试]` （不影响对话功能） |



### 6.3 输入相关边界

| 场景         | 处理方式                                   |
| ---------- | -------------------------------------- |
| 纯空格/回车发送   | 前端拦截，不发送请求                             |
| 超过 2000 字  | 输入框底部红色提示"已超过 2000 字限制"，发送按钮不可点击       |
| 快速连击发送     | 防抖处理，300ms 内只响应第一次                     |

| 粘贴超长文本     | 自动截断到 2000 字，Toast 提示"输入内容已截断至 2000 字" |



## 七、动效与过渡规范

| 场景         | 动效              | 参数                                                      |
| ---------- | --------------- | ------------------------------------------------------- |
| 对话气泡出现     | 从底部滑入 + 淡入      | `translateY(12px) → 0`, `opacity 0→1`, `300ms ease-out` |
| AI 回答流式输出  | 逐字/逐句渲染，光标闪烁    | 光标 `▍` 闪烁频率 `0.8s`，文字渲染跟随 SSE 流速                        |
| 引用标签展开     | 高度展开 + 淡入       | `max-height 0→auto`, `opacity 0→1`, `250ms ease-in-out` |
| PDF 跳转滚动   | 平滑滚动到目标位置       | `scroll-behavior: smooth`, 约 `400ms`                    |
| PDF 高亮呼吸闪烁 | 透明度脉冲 3 次后稳定    | `opacity 0.4↔1.0` 循环 3 次，每次 `600ms`，最终保持 `0.8`          |
| 旧高亮清除      | 淡出消失            | `opacity → 0`, `200ms ease-out`                         |
| 预览区折叠      | 宽度收缩 + 对话区扩展    | `width → 40px`, `300ms ease-in-out`                     |
| 预览区展开      | 宽度展开 + 对话区收缩    | `width → 默认比例`, `300ms ease-in-out`                     |
| 上传进度条      | 左到右线性填充         | 蓝色填充条，跟随实际上传进度                                          |
| 解析进度-脉冲条   | 不确定进度条左右往复      | `indeterminate` 动画，蓝色渐变条纹，`2s` 循环                       |
| Toast 消息   | 从顶部滑入 → 停留 → 滑出 | `translateY(-100%)→0`, 停留 `3s`, 滑出 `200ms`              |
| 反馈面板展开     | 从气泡底部向下滑出       | `max-height 0→auto`, `200ms ease-out`                   |
| 反馈面板收起     | 向上滑入收起          | `max-height → 0`, `150ms ease-in`                       |
| 按钮 Hover 态 | 背景色加深 + 微缩放     | `background-color` 过渡 `150ms`, `scale(1.02)`            |
| 加载骨架屏      | 渐变条纹左右扫光        | 浅灰→白→浅灰线性渐变, `1.5s` 循环                                  |
| 拖拽上传 - 拖入  | 全屏遮罩淡入 + 虚线边框呼吸 | 遮罩 `opacity 0→0.6`, `200ms`; 虚线 `border-color` 脉冲       |
| 拖拽上传 - 松手  | 遮罩缩小消失 + 弹跳确认   | `scale(1→0.95→1)` + 遮罩淡出 `200ms`                        |
| 暗色模式切换     | 全局颜色渐变过渡        | `transition: color, background-color 400ms ease`        |
