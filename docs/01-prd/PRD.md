## 一、产品概述

一个极简的 **"上传PDF → 解析切片 → 对话问答"** Web 应用。用户上传一份文档（如IC Datasheet），系统自动解析、切片、向量化，随后用户可以针对该文档进行多轮对话，回答附带原文引用溯源。



**一句话描述：** 上传一个 PDF，马上跟它聊天，每个回答都能溯源到原文。



## 二、目标用户

IC 设计/验证工程师、FAE，需要快速从一份 Datasheet 或 Design Spec 中查找参数、理解规格。



## 三、核心用户场景

| # | 场景      | 描述                                                  |
| - | ------- | --------------------------------------------------- |
| 1 | 寄存器参数查询 | 工程师上传一份 Datasheet，提问"XX寄存器的复位值是多少？"，系统返回精确参数并标注出处页码 |
| 2 | 电气特性查阅  | 提问"VDD的最大耐压是多少？"，系统从表格中提取数据并引用对应表格区域                |
| 3 | 文档快速摘要  | 上传文档后，系统自动生成核心摘要和 3 个推荐提问，帮助用户快速上手                  |



## 四、功能需求

### P0   必须实现

#### 4.1 文档上传

* 支持上传单个 PDF 文件（≤ 100MB）

* 上传后显示进度条，状态流转：`上传中 → 解析中 → 切片中 → 就绪`

* 就绪前，聊天输入框置灰，提示"文档解析中，AI 助手稍后就绪"



#### 4.2 文档解析

* 调用 PDF 解析库提取文本内容，输出为 Markdown 格式

* 基础表格提取：将 PDF 中的表格还原为 Markdown 表格

* 提取图片并保留占位符（`![图片描述](image_xx.png)`）

* 解析超时控制：单文档上限 5 分钟，超时标记失败并提示



#### 4.3 智能切片 & 向量化

* **切片策略：** 按 Markdown Header 层级切分（优先），兜底按固定 Token（512 tokens）+ 重叠（50 tokens）切分

* **元数据继承：** 每个切片记录 `chunk_id`、`page_num`、`title_path`（所属章节路径）

* **向量化：** 调用本地 Embedding 模型生成向量，存入向量数据库

* 自动过滤空切片、纯页眉页脚切片



#### 4.4 单文档对话问答

* **检索范围严格限定**当前文档的切片（按 `doc_id` 过滤），不允许跨文档召回

* **检索方式：** 向量相似度检索，返回 Top-5 切片

* **生成回答：**

  * 将 Top-K 切片作为 Context 注入 Prompt，调用本地 LLM 生成回答

  * 回答必须标注引用锚点 `[1]` `[2]`，对应引用的切片来源

  * 若相关性得分低于阈值，回答"该文档中未找到相关信息"，禁止编造

* **流式输出：** 打字机效果逐字展示

* **多轮对话：** 维护对话历史（最近 10 轮），支持追问和指代消除



#### 4.5 引用溯源

* 回答下方展示引用列表：`[1] 文档名 - 第X页 - 章节标题`

* 点击引用锚点 → 右侧 PDF 预览区自动滚动到对应页并高亮对应段落



### P1 — 体验增强

#### 4.6 自动摘要 & 推荐问题

* 文档就绪后，自动生成 ≤ 200 字的文档摘要

* 自动生成 3 个推荐提问（如"本文档描述了哪些寄存器？"）



#### 4.7 反馈机制

* 每条回答旁提供 👍 / 👎 按钮

* 点踩时弹出可选反馈标签：`回答不准确` / `引用错误` / `信息不完整`

## 九、验收标准

|   | 验收项   | 标准                                       |
| - | ----- | ---------------------------------------- |
| 1 | 上传解析  | 上传一份 ≤50 页 PDF，5 分钟内完成解析+切片+向量化，状态显示"就绪" |
| 2 | 问答准确性 | 提问 10 个文档内有明确答案的问题，≥ 8 个回答正确             |
| 3 | 引用溯源  | 每个回答至少包含 1 个引用锚点，点击可跳转到 PDF 对应页          |
| 4 | 隔离性   | 回答内容严格来自当前文档，不出现编造内容                     |
| 5 | 无知则免  | 提问文档中不存在的内容，系统回答"未找到相关信息"                |
| 6 | 流式输出  | 回答以打字机效果逐字展示，首字延迟 ≤ 3 秒（本地环境）            |
| 7 | 多轮对话  | 支持至少 3 轮连续追问，上下文连贯                       |



## 十、不做什么（明确排除）

* ❌ 不做用户登录/权限体系

* ❌ 不做多文档/多知识库管理

* ❌ 不做文件版本控制

* ❌ 不做知识图谱/标签系统

* ❌ 不做 Rerank 重排序（Top-K 向量检索即可）

* ❌ 不做安全防御（Prompt 注入检测）

* ❌ 不做运营报表/审计日志

* ❌ 不做 OCR（仅支持文本型 PDF，不支持扫描件）
