## 2. Scope / 范围
- 页面目标：
  - 定义工作台中“对话区”这块前端表面的最终行为。
- frontend surfaces / 前端交互面:
  - 欢迎态摘要
  - 推荐问题
  - 消息列表
  - 引用来源块
  - 反馈面板
  - 检索/生成状态提示
  - 历史折叠与新消息提示
  - 无结果、错误、中断消息卡
- 本文不重写整页 40/60 分栏骨架，由 `chatfile-web` 页面文档承接。

## 4. Layout And Regions / 布局与区域划分
### 4.1 Page Skeleton / 页面骨架
```text
┌──────────────────────────────────────────────┐
│  欢迎态摘要 / 推荐问题                        │
│                                              │
│                        用户消息气泡 + 时间戳  │
│  AI 头像 + AI 消息气泡 + 时间戳              │
│         ├─ 行内引用 [1][2]                   │
│         ├─ 引用来源块（默认展开）             │
│         └─ 👍 👎 📋 操作区                    │
│                                              │
│  [↑ 查看更多历史 (共 N 轮)]                  │
│  [↓ 新消息]                                  │
│  错误消息 / 无结果 / 中断提示 / 重试按钮      │
└──────────────────────────────────────────────┘
```

### 4.2 Region Responsibilities / 区域职责
- 欢迎态区域：
  - 展示摘要和 3 个推荐问题，首次发送后收起。
- 消息流区域：
  - 承载用户消息、AI 消息、时间戳、状态提示、折叠历史和新消息提示。
- 消息附属区域：
  - 引用来源块、反馈按钮、复制按钮、tooltip、反馈面板。
- 系统反馈区域：
  - 承载检索态/生成态、无结果、服务异常、网络中断和重新生成按钮。

### 4.3 First-screen And Variant Layouts / 首屏与变体布局
- 首屏 ready 态：
```text
┌─────────────────────────────────┐
│  🧠 已准备就绪！                 │
│                                 │
│  📋 文档摘要：                   │
│  ... <= 200 字摘要 ...          │
│                                 │
│  💡 你可以试着问我：              │
│  [问题1]                         │
│  [问题2]                         │
│  [问题3]                         │
└─────────────────────────────────┘
```
- 会话进行中：
  - 欢迎态收起，仅保留消息流；新消息默认自动滚到底部，除非用户手动上滚。
- 无结果态：
```text
🤖  ┌────────────────────────────────────────┐
    │ 🤷 抱歉，在当前文档中未找到相关内容。    │
    │ 💡 建议：换一种表述 / 确认信息在文档中   │
    │   👍   👎                               │
    └────────────────────────────────────────┘
```
- 中断态：
  - 在已生成消息底部追加 `⚠️ 回答已中断` 或 `⚠️ 网络中断，回答不完整`，并允许重新生成。
- 错误态：
  - 显示带 `503 / requestId / [🔄 重试]` 的错误消息卡。

## 5. User Flow / 用户流程
- 主流程：
  - 文档 ready -> 欢迎态出现 -> 点击推荐问题或手工发送 -> 进入检索态和生成态 -> 完成后显示引用和操作区。
- 分支流程：
  - stop 中断 -> 保留已生成内容 -> 可继续提问。
  - 用户上滚查看历史 -> 新消息出现时不强制跳底，显示 `[↓ 新消息]` 按钮。
  - 点击行内引用或引用列表 -> 派发到右栏预览跳转。
  - 点击 👎 -> 展开反馈面板 -> 提交或取消。
- 失败回路：
  - `503/timeout` -> 错误消息卡 -> 重试或重新生成。
  - 网络中断 -> 保留部分内容 -> 显示 `重新生成`。

## 6. API Interaction / API 交互
| Stage / 阶段 | API | Trigger / 触发条件 | Failure Handling / 失败处理 |
| --- | --- | --- | --- |
| 欢迎态加载 | `GET /api/conversations/{docId}/welcome` | 文档 ready 且消息为空 | 返回 `pending` 时显示骨架 |
| 发送问答 | `POST /api/conversations/query:stream` | 用户发送消息 | 错误态显示错误卡 |
| 反馈提交 | `POST /api/messages/{messageId}/feedback` | 点击提交反馈 | 失败时保留面板与错误 toast |
| 历史查询 | `GET /api/conversations/{docId}/messages` | 初次进入或展开历史 | 失败时保留现有消息并提示 |

## 7. Interaction Design / 交互设计
| Scenario / 场景 | Trigger / 触发条件 | State Change / 状态变化 | Feedback / Motion / 反馈与动效 | Copy / Timing / 文案与时序 | Error / Edge Handling / 异常与边界处理 |
| --- | --- | --- | --- | --- | --- |
| 欢迎态展示 | 文档 ready 且消息为空 | 展示摘要和 3 个推荐问题 | 摘要淡入 | 推荐问题首次发送后收起 | 获取失败时显示骨架或空欢迎态 |
| 用户消息渲染 | 用户发送 | 右对齐消息入列 | 气泡自底部滑入 `300ms ease-out` | 保留时间戳 |  |
| 检索态 | 发送消息后 | 输入区禁用，消息流底部出现状态提示 | 展示 `🔍 正在检索相关内容...` | 检索阶段约 0.3-1s | 若检索失败转错误卡 |
| 生成态 | 检索命中后 | AI 气泡逐 token 渲染 | 光标 `0.8s` 闪烁 | 引用占位先灰后蓝 | stop 时转中断态 |
| 引用悬停和点击 | hover / click `[1]` 或引用条目 | tooltip 出现或派发跳转事件 | tooltip 展示前 80 字摘要 | 点击后右栏联动 | 触屏设备降级为点击显示摘要 |
| 完成态 | 回答结束 | 引用块和操作区出现 | 引用块淡入，高度展开 `250ms`，反馈按钮淡入 | 输入区重新激活并聚焦 | 若无引用则不展示引用块 |
| 历史折叠 | 消息超过 10 轮 | 旧消息灰化并折叠 | 无强动效 | 顶部显示 `[↑ 查看更多历史 (共 N 轮)]` | 展开失败时保留折叠态 |
| 新消息提示 | 用户手动上滚后有新消息 | 不自动跳底 | 底部浮现 `[↓ 新消息]` 按钮 | 点击后跳到底部 | 无新消息时隐藏 |
| 无结果 | 低相关拒答 | 输出拒答消息 | 无引用块、无复制按钮 | 文案提示换表述 | 保留反馈按钮 |
| 负反馈 | 点击 👎 | 滑出反馈面板 | 面板 `200ms ease-out` 展开 | 提交成功 toast `感谢反馈，我们会持续优化` | 取消时恢复空心状态 |
| 复制 | 点击复制 | 无状态持久化 | 顶部 toast `已复制到剪贴板` | 3 秒自动消失 | 只复制纯文本，不含引用标记 |
| 服务异常 | `503/timeout` | 输出错误消息卡 | 可见 `🔄 重试` 按钮 | 保留 `requestId` | 连续 3 次失败后由外层输入区禁用 30 秒 |
| 网络中断 | SSE 断连 | 保留部分文本并标记不完整 | 显示 `⚠️ 网络中断，回答不完整` | 可点击 `重新生成` | 引用块可基于已生成内容部分展示 |
